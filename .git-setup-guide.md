# Git Setup Guide for New Projects from Boilerplate

When creating a new project from the `my-next-forge` boilerplate:

## Step 1: Clone the boilerplate
```bash
git clone git@github.com:eric-nichols-nyc/my-next-forge.git my-new-project-name
cd my-new-project-name
```

## Step 2: Remove the original remote (IMPORTANT!)
```bash
git remote remove origin
```

## Step 3: Create your new GitHub repository
Create a new empty repository on GitHub (e.g., `my-new-project-name`)

## Step 4: Add your new remote
```bash
git remote add origin git@github.com:eric-nichols-nyc/my-new-project-name.git
```

## Step 5: Verify the remote
```bash
git remote -v
```
Should only show your new repository, NOT the boilerplate.

## Step 6: Push to your new repo
```bash
git push -u origin main
```

## Alternative: Use a different remote name
If you want to keep the boilerplate remote for reference:
```bash
git remote rename origin boilerplate
git remote add origin git@github.com:eric-nichols-nyc/my-new-project-name.git
```

This way you can still fetch updates from the boilerplate but won't accidentally push to it.
